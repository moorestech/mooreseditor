# ノードグラフのプラグイン化 Phase 2 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 編集エンジン（FormView / TableView とその依存サブツリー）・スキーマ層・プラグイン契約型を `@mooreseditor/plugin-sdk` パッケージへ切り出し、`nodeEditor/` を `plugins/node-graph/` へ移動して mooreseditor 本体への直接依存をゼロにする。動作は変えない「機能不変リファクタ」。

**Architecture:** monorepo に `packages/plugin-sdk/` を新設し、編集エンジン全体を移管（フルSDK方針）。アプリ本体・プラグイン双方が SDK を参照する。`nodeEditor/` を `plugins/node-graph/` へ git mv し、本体相対 import（`../../components/...` 等）を全て `@mooreseditor/plugin-sdk` 参照へ置換する。この段階ではまだ `React.lazy` による静的 import でノードビューを動かし、ランタイムロードは Phase 3 で行う。

**Tech Stack:** pnpm workspace / TypeScript project references / React 19 / Vite / Vitest / Playwright

設計ドキュメント: `docs/superpowers/specs/2026-05-20-node-graph-plugin-design.md`

## アプリ / SDK の境界定義（本計画の最重要ルール）

**判定ルール:** `FormView` または `TableView` が（推移的に）import するモジュールは SDK へ移す。それ以外でプラグインが必要とする型・契約も SDK へ移す。

> **SDK 命名・公開 API 範囲に関する注記:** 本計画では編集エンジン全体（FormView/TableView
> を含む）を `@mooreseditor/plugin-sdk` に集約する「フル SDK 方針」を採用している。
> これにより "内部エディタコア" と "プラグイン向け公開 API 契約" が単一パッケージに混在する。
> 着手前に、`@mooreseditor/editor-core`（内部コア）と `@mooreseditor/plugin-sdk`（契約のみ）
> に分割するか、現行一本化のまま `index.ts` の public export を明示的にキュレートするかを
> チームで決定すること。どちらの場合も SDK の公開 API 表面をドキュメント化する。
> （設計ドキュメント §4-D および §3 も参照。）

| 区分             | 配置                       | 対象                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ---------------- | -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| SDK へ移管       | `packages/plugin-sdk/src/` | `libs/schema/`、`components/FormView/`、`components/TableView/`、`components/SortableRow.tsx`、編集系フック（`useArrayDataManager` `useCopyPaste` `useDebounce` `useForeignKeyData` `useSwitchFieldAutoGeneration`）、編集系ユーティリティ（`createInitialValue` `ensureUniqueAutoGeneratedUuids` `switchFieldProcessor` `vectorConverter` `generateUuid` `pathResolver` `foreignKeyResolver` `dataInitializer` `dataValidator` `deepMerge` `schemaToZod` `autoIncrement` `notifyFieldsAdded`）、`types/json.ts`、`Column` 型 |
| アプリに残す     | `apps/mooreseditor/src/`   | `App.tsx`、`main.tsx`、`viewHost/`、`pluginHost/`（Phase 3）、`components/EditorView.tsx`、`components/SearchOverlay.tsx`、`components/Sidebar.tsx`、`components/DataSidebar.tsx`、`hooks/useProject` `useSchema` `useJson`（ロジック本体）`useNestedViewScroll` `useSaveShortcut`、`utils/saveProjectData` `devFileSystem` `notification`、`vite-plugins/`                                                                                                                                                                   |
| プラグインへ移動 | `plugins/node-graph/src/`  | `nodeEditor/` 配下 68 ファイル全て                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

注: `useJson` は「`Column` 型」を SDK へ、「フック本体」をアプリに残す（フックは別フェーズで再編しない）。`Column` 型は SDK の `types` から re-export し、アプリ側 `useJson` は SDK から型を import する。

設計ドキュメント §6 の契約型を SDK に定義する。Phase 1 で作った `viewHost/types.ts` の

- [ ] **Step 1: `contract/types.ts` を作成**

`packages/plugin-sdk/src/contract/types.ts`:

```ts
import type { ReactNode } from "react";

import type { Column } from "../schema";
import type { Schema } from "../schema";

/**
 * ホストがプラグインへ提供する API。
 * プラグインはこれを通じてのみホストの状態へアクセスする。
 */
export interface HostAPI {
  /** 現在の master カラム一覧を取得する。 */
  getColumns(): Column[];
  /** master カラムを更新する。 */
  setColumns(updater: (columns: Column[]) => Column[]): void;
  /** ロード済みスキーマ。 */
  schemas: Record<string, Schema>;
  /** スキーマを名前指定でロードする。 */
  loadSchema(name: string): Promise<Schema | null>;
  /** プロジェクトディレクトリ（未オープン時 null）。 */
  projectDir: string | null;
  /** master ディレクトリ（未設定時 null）。 */
  masterDir: string | null;
  /** ホストへ未保存変更を通知する。 */
  markDirty(): void;
  /** プラグイン専用ファイルをプロジェクト配下へ保存する。 */
  saveExtraFile(relativePath: string, content: string): Promise<void>;
  /** プラグイン専用ファイルをプロジェクト配下から読み込む（無ければ null）。 */
  readExtraFile(relativePath: string): Promise<string | null>;
  /**
   * master カラム群と任意のプラグイン専用ファイルを原子的に保存する。
   *
   * PluginView.save() は「master カラム更新 + プラグイン専用ファイル保存（例:
   * nodeGraph.v1.json）」を 1 操作として完結させる必要がある。
   * saveExtraFile だけでは master JSON を永続化できないため、この API が必要。
   */
  saveProject(
    columns: Column[],
    extraFiles?: { path: string; content: string }[],
  ): Promise<void>;
}

/**
 * プラグインがホストへ公開するビュー実体。
 * ホストの ViewRegistry にタブとして登録される。
 */
export interface PluginView {
  /** ビュー本体をレンダリングする。 */
  render(): ReactNode;
  /** 保存を実行する（任意）。 */
  save?(): Promise<void>;
  /** 未保存変更があるか（任意）。 */
  isDirty?(): boolean;
  /** 検索一致要素へフォーカスする（任意）。 */
  focusSearchMatch?(element: HTMLElement | null): void;
}

/**
 * プラグイン成果物が default export するオブジェクト。
 */
export interface PluginManifest {
  /** 一意なプラグイン ID。 */
  id: string;
  /** タブ表示名。 */
  name: string;
  /** プラグインバージョン（semver）。 */
  version: string;
  /** HostAPI を受け取りビューを生成する。 */
  createView(host: HostAPI): PluginView;
}
```
